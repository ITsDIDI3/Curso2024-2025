Data and Licensing Analysis

1. Dataset Analysis

El dataset tiene varias propiedas, como el ID, nombre del parking, su localización o número de plazas. El rango y descripción de cada propiedad se detalla a continuación.

Data Overview

Atributos Descripcion Rango
ID Número de identificación único para cada parking de bicis Desde el 65.239 hasta el 9.700.000
DESC_CLASIFICACION PROBLEMA: Es un String que se repite para todos los casos "Aparcabicicletas" única posibilidad: "Aparcabicicletas"
COD_BARRIO Código que identifica el barrio donde se encuentra el parking Desde 10 a 220
BARRIO Nombre del barrio donde se encuentra el parking de bicis 127 posibles String
COD_DISTRITO Código que identifica el distrito donde se encuentra el parking De 1 a 22
DISTRITO Nombre del distrito donde se encuentra el parking de bicis 27 posibles String
ESTADO Nos dice si el parking está operativo o no Dos posibilidades: OPERATIVO O NULO
COORD_GIS_X Coordenado en el eje X de la geolocalización del parking Desde 431.000 hasta 452.000
COORD_GIS_Y Coordenado en el eje X de la geolocalización del parking De 4.464.000 hasta 4.489.000
SISTEMA_COORD Sistema de coordenadas que se utiliza para mostrar los valores de estas mismas Una posibilidad: "ETRS89"
LATITUD Número que indica la latitud del parking De 0 a 41, concentrado mayoritariamente en el 40
LONGITUD Número que indica la longitud del parking De -3.81 a 0.02, concentrado mayoritariamente en el -3
TIPO_VIA Nos indica el tipo de vía donde se encuentra el parking de bicis 25 posibilidades
NOM_VIA String que nos indica el nombre de la vía donde está el parking 61 posibilidades
NUM_VIA Número de identificación de la vía donde se encuentra el parking De 0 a 630
COD_POSTAL Código postal de la zona donde se encuentra el parking De 28001 a 28056
DIRECCION_AUX Palabra o palabras clave que funcionan como auxiliar para encontrar el parking de bicis 391 posibilidades
NDP Número de identificación de la red donde se encuentra el parking De 11.000.000 a 32.000.000, concentrado mayoritariamente en 11.000.000, 20.000.000 y 31.000.000
FECHA_INSTALACION Fecha en la que el parking de bicis ha sido inaugurado y disponible para uso Desde el Sun Dec 31 1899 23:45:16 GMT-0014 (hora estándar de Europa central) hasta el Wed Oct 02 2024 02:00:00 GMT+0200 (hora de verano de Europa central)
CODIGO_INTERNO Código interno de identificación para reconocer el parking de bicis, que puede ser un número o un String Rango númerico: 4000.000 a 4.100.000
CONTRATO_COD Código de identificación del contrato Un valor posible: "MU-21"
MODELO String que identifica el modelo del parking de bicis Un valor posible: "MU-51"

3. Resource Naming Strategy

URI para los términos ontológicos

Se usará la siguiente URI para las clases: https://smartcity.linkeddata.es/parkingbicismadrid/ontology/{ontologyName}#{nombreClase}

Se usará la siguiente URI para las propiedades: https://smartcity.linkeddata.es/parkingbicismadrid/ontology/{ontologyName}#{nombrePropiedad}

URI para las entidades concretas

Se usará la siguiente URI para las entidades: https://smartcity.linkeddata.es/parkingbicismadrid/resource/{className}/{id}
