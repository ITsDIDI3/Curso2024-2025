Definición de la Estrategia de Nombramiento de Recursos (Resource Naming Strategy)

Para la ontología y los datos que se generen a partir del dataset, se sigue una estrategia clara de nombramiento de recursos. A continuación, se definen los principios clave que se aplicarán para las URIs de los recursos.

1. Forma de las URIs (URI Form)

Las URIs se construirán de acuerdo al tipo de recurso, siguiendo estas dos convenciones:

  • Para los términos ontológicos (clases y propiedades), se utilizarán hash URIs con el formato https://{dominio}/{path}/ontology/{NombreOntologia}#{nombreClase} para clases o https://{dominio}/{path}/ontology/{NombreOntologia}#{nombrePropiedad} para las propiedades. Esto asegura que todos los términos dentro de una clase ontológica estén claramente definidos bajo un mismo espacio de nombres.
  • Para los individuos o instancias (entidades concretas), se utilizarán slash URIs con el formato https://{dominio}/{path}/resource/{tipo-de-recurso}/{identificador}, asegurando la identificación única de cada instancia específica.

2. Negociación de Contenidos (Content Negotiation)

Se utilizará la negociación de contenido para que las URIs respondan a diferentes formatos de datos según las preferencias del cliente. Esto permitirá que las URIs devuelvan datos en formatos como:

  • HTML (para visualización web)
  • RDF/XML (para intercambio de datos en la web semántica)
  • JSON-LD (para representar datos en formato JSON en un contexto Linked Data)

3. Dominio para las URIs

Un dominio sugerido podría ser https://websemantica.net, ya que estamos en un caso académico para la asignatura Semantic Web, Linked Data and Knowledge Graphs

4. Path Structure

La Path Structure se diseñará para distinguir claramente entre los términos ontológicos y las instancias de la ontología tal y como se explicó en el punto 2.

  • Para los términos ontológicos (clases y propiedades)

    Ejemplo de URI para la clase 'Camara': https://websemantica.net/Group03/ontology/CamarasBcn#Camara

    Ejemplo de URI para la propiedad de las cámaras 'hasFechaAlta': https://websemantica.net/Group03/ontology/CamarasBcn#hasFechaAlta

  • Para las instancias individuales (como cámaras específicas, barrios, y distritos):

    Ejemplo de URI para una cámara específica con ID '1234': https://websemantica.net/Group03/resource/Camara/1234

    Ejemplo de URI para una barrio específico con código '34': https://websemantica.net/Group03/resource/Barrio/34

5. Patrones de Identificadores

Los identificadores en las URIs deben seguir los códigos ya presentes en el dataset.
